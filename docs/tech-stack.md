# 纸片人女友 2.0 技术栈

## 1. 核心框架

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| 前端与应用框架 | Next.js (App Router) | 全栈框架，SSR/SSG/API Routes 一体 |
| 语言 | TypeScript | 全项目统一使用 TypeScript |

## 2. 数据库

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| 数据库 | Neon | Serverless Postgres，Vercel 原生集成，冷启动快 |
| 数据库访问 | 手写 SQL + postgres.js | 直觉、无额外抽象，后续可按需引入 Drizzle |

## 3. 认证

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| 认证库 | BetterAuth | 现代化 Auth 库，配置简单 |
| 登录方式 | 邮箱注册 + 邮箱登录 | 第一版必须实现 |
| 社交登录 | 预留，P1 阶段接入 | BetterAuth 原生支持，后续可快速开启 |

## 4. UI

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| CSS 方案 | Tailwind CSS | 原子化 CSS，与 Next.js 生态深度集成 |
| 组件 | 自定义设计 | 微信风格聊天界面，不依赖组件库 |
| 图标 | Lucide React | 轻量、风格统一 |

## 5. 实时通信

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| 消息推送 | SSE (Server-Sent Events) | 适合 AI 流式输出，单向推送，实现简单 |

## 6. 文件存储

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| 对象存储 | Cloudflare R2 | S3 兼容、无出站流量费、成本低 |
| 存储内容 | 角色基准图、生成图片、语音文件、用户头像 | |

## 7. AI 能力层

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| API 提供商 | 火山引擎（火山方舟） | 统一平台，LLM / TTS / 图像生成一站式 |
| LLM | 开发收尾阶段选定 | 候选：Doubao-pro-32k 等，需在 Postman 验通后确认 |
| TTS | 开发收尾阶段选定 | 候选：豆包语音合成 2.0，需在 Postman 验通后确认 |
| 图像生成 | 开发收尾阶段选定 | 候选：DreamO (seededit-3.0-i2i)，需支持图生图模式，需在 Postman 验通后确认 |

### Provider 抽象层要求

代码层必须预留可替换的 Provider 抽象，具体模型选择不影响业务逻辑代码。开发阶段可先用 mock 数据跑通主链路，收尾阶段替换为真实 API。

## 8. 部署与托管

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| 代码托管 | GitHub | 从第一个 commit 开始管理 |
| 部署平台 | Vercel | 与 Next.js + Neon 原生集成 |
| 域名 | Vercel 默认域名，P2 阶段可购买独立域名 | |

## 9. 开发工具

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| 包管理器 | pnpm | 快、省磁盘空间 |
| 代码规范 | ESLint + Prettier | 统一代码风格 |
| API 调试 | Postman | 先验通每个外部 API 再接入代码 |
